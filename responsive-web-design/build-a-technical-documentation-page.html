Introduction

Node.js is a JavaScript runtime built on Chrome’s V8 engine. It lets you run JavaScript outside the browser to build servers, tools, and scripts.

Its event-driven, non-blocking I/O model makes it well-suited for I/O-heavy applications, CLIs, and microservices.

What you should already know

This guide assumes a basic understanding of the command line and JavaScript syntax.

  • Comfort with running commands like node -v and npm -v
  • Basic JS (variables, functions, modules)
  • A code editor (VS Code, etc.)

You can still follow along if you are new; just copy the examples and experiment.

Hello world

Write a simple script and run it with Node.

// hello.js
console.log("Hello, World!");

// run in your terminal
// node hello.js

If you see the text printed in your terminal, your setup works.

Modules

Node supports CommonJS with require() and ES Modules with import.

// CommonJS
const os = require("os");
console.log(os.platform());

// ES Modules (package.json → { "type": "module" })
import os from "os";
console.log(os.platform());

Pick one style and keep it consistent within a project.

npm and scripts

npm installs packages and runs project scripts defined in package.json.

npm init -y
npm install express

Add scripts to automate tasks like starting your app or running a dev server.

Asynchronous

Use callbacks, Promises, or async/await to perform non-blocking operations.

import { readFile } from "fs/promises";

const text = await readFile("file.txt", "utf8");
console.log(text);

Prefer Promises and async/await for readable control flow.

File system

The fs module lets you read and write files and directories.

import { mkdirSync, writeFileSync } from "fs";

mkdirSync("logs", { recursive: true });
writeFileSync("logs/app.log", "started");
HTTP server

Create a simple server with the built-in http module.

import http from "http";

const server = http.createServer((req, res) => {
  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("OK");
});
server.listen(3000);

Visit http://localhost:3000 in your browser to test it.

Environment variables

Use environment variables to store configuration like ports and secrets.

# .env
PORT=3000

Load them with a library such as dotenv or your platform’s secret manager.
